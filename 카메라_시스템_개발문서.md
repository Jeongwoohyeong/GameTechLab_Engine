## 1. 카메라 시스템 구조

### 1.1 아키텍처 개요

카메라 시스템은 **PlayerCameraManager**를 중심으로 **Camera Modifier** 패턴을 사용하여 구현되어 있습니다.

```
PlayerCameraManager
├── Camera / CameraComponent (실제 카메라)
├── ViewTarget (카메라가 추적할 대상)
├── Camera Modifiers (우선순위 기반 스택)
│   ├── CameraModifier_CameraShake (Priority: 100)
│   ├── CameraModifier_CameraTransition (Priority: 150)
│   └── CameraModifier_CameraFadeInOut (Priority: 240)
├── Spring Arm System (충돌 감지 카메라 팔로우)
├── Letter Box System (시네마틱 바)
└── Fade System (화면 페이드)
```

### 1.2 주요 클래스

**파일 위치:**
- `Engine/Source/Manager/Camera/Public/PlayerCameraManager.h`
- `Engine/Source/Manager/Camera/Private/PlayerCameraManager.cpp`

**주요 멤버 변수:**
```cpp
// 카메라
UCamera* Camera;                        // 에디터 모드 카메라
UCameraComponent* CameraComponent;      // PIE/게임 모드 카메라

// 뷰 타겟
FViewTarget ViewTarget;                 // 카메라가 추적할 대상

// 모디파이어
TArray<UCameraModifier*> ModifierList;  // 우선순위 기반 정렬
```

### 1.3 카메라 뷰 타입

```cpp
enum class ECameraViewType : uint8
{
    ThirdPerson,     // 3인칭 (기본)
    FirstPerson,     // 1인칭
    Cinematic,       // 시네마틱
    DeathCam,        // 사망 카메라
    FreeCam          // 자유 카메라
};
```

## 2. Camera Modifier 패턴

### 2.1 개념

Camera Modifier는 카메라에 다양한 효과를 스택 형태로 적용할 수 있는 시스템입니다. 각 모디파이어는 우선순위를 가지며, 낮은 우선순위부터 순차적으로 적용됩니다.

**파일 위치:**
- `Engine/Source/Manager/Camera/Public/CameraModifier.h`

### 2.2 베이스 클래스 구조

```cpp
class UCameraModifier : public UObject
{
protected:
    APlayerCameraManager* CameraOwner;  // 소유 카메라 매니저
    float AlphaInTime;                   // 블렌드 인 시간
    float AlphaOutTime;                  // 블렌드 아웃 시간
    float Alpha;                         // 현재 알파 (0~1)
    float TargetAlpha;                   // 목표 알파
    bool bDisabled;                      // 비활성화 여부
    uint8 Priority;                      // 우선순위 (낮을수록 먼저)

public:
    // 카메라 수정
    virtual bool ModifyCamera(float DeltaTime,
                             FVector& CameraLocation,
                             FRotator& CameraRotation);

    // 모디파이어 상태 업데이트
    virtual void UpdateModifier(float DeltaTime);
};
```

### 2.3 모디파이어 우선순위

```cpp
Priority 100: Camera Shake         // 가장 먼저 적용 (흔들림)
Priority 150: Camera Transition    // 카메라 전환
Priority 240: Fade In/Out          // 마지막 적용 (페이드)
```

### 2.4 모디파이어 적용 흐름

```cpp
void APlayerCameraManager::ApplyCameraModifiers(
    float DeltaTime,
    FVector& CameraLocation,
    FRotator& CameraRotation)
{
    // 우선순위 순서대로 모디파이어 적용
    for (UCameraModifier* Modifier : ModifierList)
    {
        if (Modifier && !Modifier->IsDisabled())
        {
            Modifier->UpdateModifier(DeltaTime);
            Modifier->ModifyCamera(DeltaTime, CameraLocation, CameraRotation);
        }
    }
}
```

## 3. Camera Shake (카메라 흔들림)

### 3.1 개요

카메라 흔들림은 폭발, 충격 등의 타격감을 표현하는 데 사용됩니다. Bezier 곡선을 사용한 감쇠(Decay) 커브를 지원합니다.

**파일 위치:**
- `Engine/Source/Manager/Camera/Public/CameraModifier_CameraShake.h`
- `Engine/Source/Manager/Camera/Private/CameraModifier_CameraShake.cpp`

### 3.2 클래스 구조

```cpp
class UCameraModifier_CameraShake : public UCameraModifier
{
    // Bezier Decay Curve Settings
    float BezierCP[4] = { 0.250f, 0.460f, 0.450f, 0.940f };  // easeOutQuad
    bool bUseBezierDecay = true;

private:
    float ShakeIntensity;      // 흔들림 강도
    float ShakeDuration;       // 지속 시간
    float ShakeTimer;          // 경과 시간
    FVector ShakeOffset;       // 현재 오프셋
    bool bIsShaking;           // 진행 중 여부
};
```

### 3.3 사용법

```cpp
// 카메라 흔들림 시작
APlayerCameraManager* CamMgr = GetPlayerCameraManager();
CamMgr->StartCameraShake(
    5.0f,   // Intensity (강도)
    0.5f    // Duration (지속 시간, 초)
);
```

### 3.4 Bezier Decay 커브

흔들림은 시간에 따라 점점 약해집니다. Bezier 곡선을 사용하여 부드러운 감쇠 효과를 구현합니다.

```cpp
void UCameraModifier_CameraShake::UpdateModifier(float DeltaTime)
{
    float t = ShakeTimer / ShakeDuration;  // [0, 1]
    float ShakeAlpha;

    if (bUseBezierDecay)
    {
        // Bezier 커브 사용
        float BezierT = EvaluateBezier(t, BezierCP);
        ShakeAlpha = 1.0f - BezierT;  // 1.0에서 0.0으로 감소
    }
    else
    {
        // 선형 감소
        ShakeAlpha = 1.0f - t;
    }

    float CurrentIntensity = ShakeIntensity * ShakeAlpha;

    // 무작위 오프셋 생성
    ShakeOffset.X = ((float)rand() / RAND_MAX - 0.5f) * 2.0f * CurrentIntensity;
    ShakeOffset.Y = ((float)rand() / RAND_MAX - 0.5f) * 2.0f * CurrentIntensity;
    ShakeOffset.Z = ((float)rand() / RAND_MAX - 0.5f) * 2.0f * CurrentIntensity;
}
```

### 3.5 커스텀 Bezier 커브

```cpp
// 커스텀 감쇠 커브 설정 (에디터에서)
UCameraModifier_CameraShake* ShakeMod =
    CamMgr->FindModifier<UCameraModifier_CameraShake>();

if (ShakeMod)
{
    // easeOutCubic: { 0.215, 0.610, 0.355, 1.0 }
    ShakeMod->BezierCP[0] = 0.215f;
    ShakeMod->BezierCP[1] = 0.610f;
    ShakeMod->BezierCP[2] = 0.355f;
    ShakeMod->BezierCP[3] = 1.0f;
}
```

## 4. Camera Transition (카메라 전환)

### 4.1 개요

카메라를 특정 위치/회전 또는 특정 액터로 부드럽게 전환합니다. 다양한 Easing 함수와 Bezier 곡선을 지원합니다.

**파일 위치:**
- `Engine/Source/Manager/Camera/Public/CameraModifier_CameraTransition.h`
- `Engine/Source/Manager/Camera/Private/CameraModifier_CameraTransition.cpp`

### 4.2 클래스 구조

```cpp
class UCameraModifier_CameraTransition : public UCameraModifier
{
    // Bezier 곡선 컨트롤 포인트
    float BezierCP[4] = { 0.250f, 0.460f, 0.450f, 0.940f };

private:
    FViewTarget TransitionStartView;    // 시작 뷰
    FViewTarget TransitionTargetView;   // 목표 뷰
    float TransitionTimer;              // 진행 시간
    float TransitionDuration;           // 지속 시간
    ECameraEaseType TransitionEaseType; // Easing 타입
    bool bIsTransitioning;              // 전환 중 여부

};
```

### 4.3 사용법

#### 특정 위치/회전으로 전환

```cpp
APlayerCameraManager* CamMgr = GetPlayerCameraManager();

FVector TargetLocation(100.0f, 200.0f, 150.0f);
FRotator TargetRotation(0.0f, 90.0f, 0.0f);

CamMgr->StartTransitionToLocation(
    TargetLocation,
    TargetRotation,
    2.0f,                           // Duration (2초)
```

#### 특정 액터로 전환

```cpp

FVector Offset(-200.0f, 0.0f, 100.0f);  // 액터 뒤쪽 100 위

    TargetActor,

    ECameraEaseType::SmoothStep,    // Easing 타입
```

#### 커스텀 Bezier 커브 사용

```cpp

float CustomBezier[4] = { 0.42f, 0.0f, 0.58f, 1.0f };  // easeInOut

    TargetRotation,

    -1.0f,              // FOV 변경 없음

);
```

## 5. Fade In/Out (페이드 효과)

### 5.1 개요

화면을 특정 색상으로 페이드 아웃/인 하는 효과입니다. 3단계 페이드 시스템을 지원합니다.

**파일 위치:**
- `Engine/Source/Manager/Camera/Public/CameraModifier_CameraFadeInOut.h`
- `Engine/Source/Manager/Camera/Private/CameraModifier_CameraFadeInOut.cpp`

### 5.2 클래스 구조

```cpp
class UCameraModifier_CameraFadeInOut : public UCameraModifier
{
private:
    enum class EFadeStage : uint8
    {
        None,     // 비활성
        FadeOut,  // 페이드 아웃 (화면 → 색상)
        Hold,     // 유지 (완전 암전)
        FadeIn    // 페이드 인 (색상 → 화면)
    };

    FVector4 FadeColor;         // 페이드 색상
    float FadeOutDuration;      // 페이드 아웃 시간
    float FadeInDuration;       // 페이드 인 시간
    float HoldDuration;         // 유지 시간
    float StageTimer;           // 현재 단계 타이머
    bool bIsFading;             // 페이드 중 여부
    EFadeStage CurrentStage;    // 현재 단계
};
```

### 5.3 사용법

#### 기본 페이드 (단순 페이드 아웃/인)

```cpp
APlayerCameraManager* CamMgr = GetPlayerCameraManager();

// 검은색으로 페이드 아웃
CamMgr->StartCameraFade(
    1.0f,                           // Duration (1초)
    FVector4(0, 0, 0, 1),          // Color (검은색)
    true,                           // bFadeOut (페이드 아웃)
    false                           // bHoldWhenFinished (유지 안 함)
);

// 페이드 인
CamMgr->StartCameraFade(
    1.0f,
    FVector4(0, 0, 0, 1),
    false,                          // bFadeOut (페이드 인)
    false
);
```

#### 3단계 페이드 (페이드 아웃 → 유지 → 페이드 인)

```cpp
// 페이드 아웃 → 2초 유지 → 페이드 인
CamMgr->StartFadeInOut(
    1.0f,                           // FadeOutDuration (1초)
    1.0f,                           // FadeInDuration (1초)
    FVector4(0, 0, 0, 1),          // FadeColor (검은색)
    2.0f                            // HoldDuration (2초 유지)
);
```

#### 흰색 페이드 (화이트 아웃)

```cpp
CamMgr->StartFadeInOut(
    0.5f,                           // 빠른 페이드 아웃
    1.5f,                           // 느린 페이드 인
    FVector4(1, 1, 1, 1),          // 흰색
    0.3f                            // 짧은 유지
);
```

### 5.4 페이드 중단

```cpp
// 점진적 중단 (페이드 인으로 전환)
CamMgr->StopFadeInOut(false);

// 즉시 중단 (화면 복구)
CamMgr->StopFadeInOut(true);
```

### 5.5 페이드 상태 확인

```cpp
float FadeAmount = CamMgr->GetFadeAmount();  // 0 = 투명, 1 = 완전 페이드

```

## 6. Spring Arm (스프링 암)

### 6.1 개요

Spring Arm은 카메라가 플레이어를 부드럽게 따라가면서 벽이나 장애물과 충돌 시 자동으로 가까워지는 시스템입니다.

**파일 위치:**
- `Engine/Source/Manager/Camera/Private/PlayerCameraManager.cpp` (UpdateSpringArm 함수)

### 6.2 주요 기능

- **부드러운 카메라 팔로우**: Lerp를 사용한 보간
- **충돌 감지**: AABB 기반 Line-Trace
- **자동 거리 조절**: 충돌 시 카메라가 타겟에 가까워짐

### 6.3 설정

```cpp
APlayerCameraManager* CamMgr = GetPlayerCameraManager();

// Spring Arm 파라미터 설정
CamMgr->SetSpringArmParams(
    FVector(-300.0f, 0.0f, 100.0f),  // Offset (뒤 300, 위 100)
    300.0f,                           // ArmLength (카메라 거리)
    10.0f                             // InterpSpeed (따라가는 속도)
);

// Spring Arm 활성화/비활성화
CamMgr->SetSpringArmEnabled(true);

// 충돌 감지 활성화/비활성화
CamMgr->SetSpringArmCollisionEnabled(true);
```

### 6.4 충돌 감지 알고리즘

```cpp
void APlayerCameraManager::UpdateSpringArm(float DeltaTime)
{
    // 목표 위치 계산
    FVector DesiredLocation = CalculateCameraLocationForView(CurrentViewType);

    // 현재 위치에서 목표 위치로 Lerp
    FVector NewLocation = Lerp(CurrentLocation, DesiredLocation,
                               SpringArmInterpSpeed * DeltaTime);

    // 충돌 감지 (AABB Line-Trace)
    if (bEnableCollisionTest)
    {
        // ShapeComponent와의 충돌 체크
        // 충돌 감지 시 카메라를 타겟에 가깝게 당김
        if (bHitDetected)
        {
            float SafeDistance = max(MinHitDistance - 10.0f, 0.0f);
            FVector SafeLocation = LineStart + TraceDirection * SafeDistance;
            NewLocation = Lerp(CurrentLocation, SafeLocation,
                              SpringArmInterpSpeed * DeltaTime * 2.0f);
        }
    }

    // 카메라 위치 업데이트
    Camera->SetLocation(NewLocation);
}
```

## 8. Hit Stop (히트 스톱)

### 8.1 개요

Hit Stop은 타격 순간 시간을 짧게 정지시켜 타격감을 극대화하는 효과입니다.

**파일 위치:**
- `Engine/Source/Manager/Time/Public/TimeManager.h`
- `Engine/Source/Manager/Time/Private/TimeManager.cpp`

### 8.2 사용법

```cpp
UTimeManager* TimeMgr = UTimeManager::GetSingleton();

// Hit Stop 시작 (0.1초 정지)
TimeMgr->StartHitStop(0.1f);

// 상태 확인
bool bIsActive = TimeMgr->IsHitStopActive();
```

### 8.3 구현 원리

```cpp
void UTimeManager::UpdateTimeDilation()
{
    // Hit Stop 업데이트 (Slomo보다 우선)
    if (HitStopTimeRemaining > 0.0f)
    {
        HitStopTimeRemaining -= RawDeltaTime;

        if (HitStopTimeRemaining <= 0.0f)
        {
            // Hit Stop 종료 - 이전 상태 복구
            if (SlowMotionTimeRemaining > 0.0f)
                GlobalTimeDilation = SlowMotionSpeed;
            else
                GlobalTimeDilation = 1.0f;
        }
        else
        {
            // Hit Stop 활성 - 시간 정지

    }

}
```

## 9. Slow Motion (슬로우 모션)

### 9.1 개요

Slow Motion은 게임 시간을 느리게 만들어 극적인 연출이나 집중 효과를 제공합니다.

**파일 위치:**

- `Engine/Source/Manager/Time/Private/TimeManager.cpp`

### 9.2 사용법

```cpp

    0.3f,   // Speed (30% 속도)
    2.0f    // Duration (2초 동안 유지)
);

```

### 9.3 구현 원리

```cpp
void UTimeManager::StartSlowMotion(float Speed, float Duration)
{
    // 현재 Time Dilation 저장
    if (SlowMotionTimeRemaining <= 0.0f)
    {
        PreviousTimeDilation = GlobalTimeDilation;
    }

    SlowMotionSpeed = Clamp(Speed, 0.0f, 1.0f);
    SlowMotionTimeRemaining = Duration;
    GlobalTimeDilation = SlowMotionSpeed;
}

void UTimeManager::UpdateTimeDilation()
{
    // Slow Motion 업데이트
    if (SlowMotionTimeRemaining > 0.0f)
    {
        SlowMotionTimeRemaining -= RawDeltaTime;

        if (SlowMotionTimeRemaining <= 0.0f)
        {
            // 종료 - 이전 속도로 복구
            GlobalTimeDilation = PreviousTimeDilation;
        }
        else
        {
            GlobalTimeDilation = SlowMotionSpeed;
        }
    }
}
```

## 10. Easing Functions (이징 함수)

### 10.1 개요

Easing 함수는 카메라 전환, 페이드 등에서 부드러운 애니메이션을 만들기 위한 보간 함수입니다.

**파일 위치:**
- `Engine/Source/Global/Function.h`

### 10.2 지원하는 Easing 타입

```cpp
enum class ECameraEaseType : uint8
{
    Linear,        // 선형 보간 (일정한 속도)
    EaseIn,        // 천천히 시작 (가속)
    EaseOut,       // 천천히 끝 (감속)
    EaseInOut,     // 천천히 시작하고 끝 (S자 곡선)
    SmoothStep,    // 부드러운 시작과 끝 (Cubic)
    SmootherStep,  // 더 부드러운 시작과 끝 (Quintic)
    Bezier         // 커스텀 Bezier 곡선
};
```

### 10.3 각 Easing 함수 공식

```cpp
inline float ApplyEasing(float Alpha, ECameraEaseType EaseType,
                         const float* BezierCP = nullptr)
{
    Alpha = Clamp(Alpha, 0.0f, 1.0f);

    switch (EaseType)
    {
    case ECameraEaseType::Linear:
        return Alpha;

    case ECameraEaseType::EaseIn:
        // Quadratic ease in: y = x²
        return Alpha * Alpha;

    case ECameraEaseType::EaseOut:
        // Quadratic ease out: y = x(2-x)
        return Alpha * (2.0f - Alpha);

    case ECameraEaseType::EaseInOut:
        // Quadratic ease in-out
        if (Alpha < 0.5f)
            return 2.0f * Alpha * Alpha;
        else
            return 1.0f - pow(-2.0f * Alpha + 2.0f, 2.0f) / 2.0f;

    case ECameraEaseType::SmoothStep:
        // Cubic smooth step: y = x²(3-2x)
        return Alpha * Alpha * (3.0f - 2.0f * Alpha);

    case ECameraEaseType::SmootherStep:
        // Quintic smooth step: y = x³(x(6x-15)+10)
        return Alpha * Alpha * Alpha *
               (Alpha * (Alpha * 6.0f - 15.0f) + 10.0f);

    case ECameraEaseType::Bezier:
        // Cubic Bezier curve
        if (BezierCP != nullptr)
            return CameraEasing::EvaluateBezier(Alpha, BezierCP);
        else
            return Alpha;  // Fallback
    }
}
```

### 10.5 Bezier 곡선

Bezier 곡선은 4개의 제어점을 사용하여 커스텀 곡선을 만듭니다.

```cpp
// 제어점: [P1.x, P1.y, P2.x, P2.y]
// P0 = (0, 0), P3 = (1, 1)은 고정

// easeOutQuad (기본값)
float easeOutQuad[4] = { 0.250f, 0.460f, 0.450f, 0.940f };

// easeInOutCubic

// easeOutElastic (탄성)

```

#### Bezier 곡선 평가 (Newton-Raphson Method)

```cpp
namespace CameraEasing
{
    inline float EvaluateBezier(float x, const float P[4])
    {
        // Newton-Raphson으로 t 찾기 (BezierX(t) = x)
        float t = x;
        for (int i = 0; i < 8; i++)
        {
            float currentX = BezierX(t, P);
            float diff = currentX - x;

            if (fabsf(diff) < 0.001f)
                break;

            float dxdt = /* X의 미분 */;
            t = t - diff / dxdt;
            t = Clamp(t, 0.0f, 1.0f);
        }

        return BezierY(t, P);
    }
}
```

```cpp
void PlayBossIntroSequence(AActor* BossActor)
{
    UTimeManager* TimeMgr = UTimeManager::GetSingleton();
    APlayerCameraManager* CamMgr = GetPlayerCameraManager();

    // 1. Letter Box 시작 (시네마틱 모드)
    CamMgr->StartLetterBox(0.12f, 1.0f);

    // 2. 보스로 카메라 전환 (SmoothStep)
    FVector BossOffset(-500.0f, 0.0f, 200.0f);  // 보스 전면
    CamMgr->StartTransitionToActor(
        BossActor,
        2.0f,
        ECameraEaseType::SmoothStep,
        BossOffset
    );

    // 3. 슬로우 모션 시작
    TimeMgr->StartSlowMotion(0.5f, 3.0f);

    // 4. 2초 후 보스 클로즈업
    Timer::SetTimeout([=]() {
        FVector CloseupOffset(-200.0f, 100.0f, 100.0f);
        CamMgr->StartTransitionToActor(
            BossActor,
            1.5f,
            ECameraEaseType::EaseInOut,
            CloseupOffset
        );
    }, 2.0f);

    // 5. 4초 후 전투 시작 (Letter Box 종료, 일반 속도)
    Timer::SetTimeout([=]() {
        CamMgr->StopLetterBox(1.0f);
        TimeMgr->StopSlowMotion();

        // 플레이어 카메라로 복귀
        AActor* Player = GetPlayerCharacter();
        CamMgr->SetViewTarget(Player);
    }, 4.0f);
}
```
